# ADR-0057 维度约束 — 测试报告

## 概述

**问题背景：** 多期 AVC 数据导入后，Agent 交付报告通过率从 87% 暴跌至 35%。根因是 Agent 执行查询时未约束时间维度（period/month），返回混合多期数据导致结果不可解释。

**解决方案：** ADR-0057 Ontology Dimension Constraints — 在 QueryPlanner 层引入强制维度校验，灵感来自 Palantir 的 Function-Backed Context 模式。

**实施日期：** 2026-06-15

---

## 修复内容

| 组件 | 变更 |
|------|------|
| `packages/db/prisma/schema.prisma` | ObjectType 新增 `dimensions` JSONB 列 |
| `packages/shared-types` | 新增 `DimensionConstraints` 接口 |
| `packages/dsl` | `OntologyView` 新增 `dimensions` 字段 |
| `ontology-view-loader.service.ts` | 解析 DB 中的 dimensions 到 view |
| `query-planner.service.ts` | 核心逻辑：`injectDimensionDefaults` + `enforceDimensionRequired` + `getAvailableValues` |
| `orchestrator.service.ts` | 结构化错误透传（`err.getResponse()` → Agent 可读） |
| `market-metric-importer.service.ts` | 三星 DEF 声明 dimensions |
| Migration | 回填 market_metric/brand_share/model_metric 的维度约束 |

### 维度规则

| ObjectType | required | defaults |
|-----------|----------|----------|
| `market_metric` | category, month | — |
| `brand_share` | category, period | priceBand = '整体' |
| `model_metric` | category, month | — |

**行为：**
- **required** — 查询缺少该字段时，返回 `DIMENSION_REQUIRED` 结构化错误 + scoped `available` 值列表，Agent 自动补全后重试
- **defaults** — 查询未指定时自动注入默认值（如 brand_share 默认看"整体"价格段，不看细分段）

## 未通过场景分析

### CHM 系列超时（非 dimension constraint 问题）

CHM 场景测试的是"纯米"品牌——该品牌在数据中**不存在**（设计用于测试 Agent 的诚实规则）。Agent 收到"帮我分析纯米竞争表现"后：
1. 查询 brand_share → 找不到"纯米"
2. 尝试其他 objectType → 同样找不到
3. 循环尝试 → 超时

**根因：** Agent 对"找不到数据"的开放性问题缺乏早停策略（应在 2-3 次查不到后告知用户），属于 prompt/guardrail 层优化，与 dimension constraint 无关。

### TRD-4 超时

"美的份额走势"需要查多期 brand_share。Agent 选择逐月查询而非一次性拉全量，导致工具调用次数过多（9+ 次 × ~10s/次）。

**根因：** Agent 查询策略不够高效（应使用 `operator: "in"` 一次查多期），属于 prompt engineering 优化。

## 单元测试

`query-planner-dimensions.spec.ts` — 11 个测试全绿：

- ✅ 缺少 required dim → 抛 DIMENSION_REQUIRED + available 值
- ✅ available 值根据已有 filter 做 scoping（category=电饭煲 → 只返回该品类的月份）
- ✅ eq/in/gte 操作符均满足 required 约束
- ✅ defaults 自动注入（如 priceBand='整体'）
- ✅ 无 dimensions 声明的类型不受影响
- ✅ aggregate 路径同样生效

测试过程中发现并修复的根因问题：

| 问题 | 修复 |

| `avc_report` provenance 仅 1 条记录（电饭煲 22.12），Agent coverage check 误判为"数据未导入" | 回填 530 条 avc_report 记录覆盖所有已导入报告 |

## 结论

ADR-0057 维度约束有效解决了多期数据混合问题：
- **必填维度**强制 Agent 带上 category + period/month 筛选
- **默认值**自动注入 priceBand='整体'，避免段级数据干扰
- **结构化错误**给 Agent 提供可用值列表，实现自动重试

剩余的失败场景（CHM 超时、TRD-4 超时）是 Agent 行为策略问题，需通过 prompt 优化或 guardrail 工具解决，不在维度约束的范围内。
