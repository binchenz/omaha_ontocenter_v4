## 一句话结论

**核心三件事（数据查询 / 数据分析 / 图表）方向正确、多轮上下文出色、越界稳健不幻觉——但有两个会击穿信任的正确性缺陷 + 一条 30–60s 的时延长尾，目前"可用于有人盯着的场景，不可用于无人监督的自助场景"。**

这恰好印证了你的判断：用户实际只用 query/analysis/chart 三件事，而这三件正是 chat 当前真正打磨过的能力（ADR-0063 把 chat 定性为 runtime 分析 Agent 是对的）；"agent 能做所有环节"的设计在 consume 面根本没被触发。

## 🔴 必须修的缺陷

### 缺陷 1：零售额数字 10× 不稳定（信任击穿 · P0）

`零售额` 这个**最高频指标**会被非确定性地报错一个数量级：

| 出处 | 报出值 | Ground truth | 判定 |
|---|---|---|---|
| 冒烟测试（同问） | 39,012.84 万元 | 39,012.84 万元 | ✓ |
| **A1** | **3,901.28 万元** | 39,012.84 万元 | ✗ ÷10 |
| **C1 轮2** | **3,901.28 万元** | 39,012.84 万元 | ✗ ÷10 |
| C1 轮1 | 39,012.84 万元 | 39,012.84 万元 | ✓ |
| C1 轮3 / D1 / D3 | 正确 | — | ✓ |

**关键证据**：C1 在**同一段对话内**先对（轮1=39,012.84）、再错（轮2=3,901.28）、又对（轮3）。tool 返回的原始 value 三轮完全相同（39012.844851），所以这**不是取数错误，是 LLM 把 5 位数字转写进散文时丢了一位**（39012.84→3901.28 正好 ÷10）。

**为什么严重**：① 零售额是用户问得最多的指标；② 非确定性，~30% 概率触发；③ 用户无从分辨对错；④ 数据本身是对的，纯属格式化幻觉。

**建议修法**：不要让 LLM 转写大数字。在 `query_objects`/`aggregate_objects` 的结果里附一个**服务端预格式化的 `displayValue`**（"39,012.84 万元 / 3.90 亿元"），prompt 里要求"金额必须原样引用 displayValue，不得自行换算或重排数字"。这是把数值可靠性从 LLM 手里收回到确定性代码里。

### 缺陷 2：market_metric 月度覆盖被误判为"只有年度快照"（P1）

**B2「画电饭煲零售额趋势折线图」整轮翻车**：12 次工具调用（撞满 `MAX_TOOL_ITERATIONS=12` 硬顶）、**0 图表**，最后答"25.07–25.11 无数据，数据集中在每年12月"。

**Ground truth：电饭煲零售额在 `market_metric` 里有连续 53 个月覆盖（21.12 → 26.04，25.07–25.11 全部存在）**。这是**确信地答错眼前就有的数据**——本battery最差答案。

**根因**：Agent 把 `brand_share`/`avc_report` 的 5 个年度报告期（22.12/23.12/24.12/25.12/26.04）误当成 `market_metric` 的覆盖期，于是反复查错维度、耗尽预算、幻觉缺数。E4「帮我分析电饭煲」也犯同病（"市场规模趋势暂未查得…报告覆盖范围限制"），但靠份额数据兜了底。

**建议修法**：skill prose 里明确点出两套不同的时间粒度——`market_metric.month` 是**连续月度**（21.12–至今），`brand_share.period`/`avc_report` 是**稀疏年度快照**；查趋势/月度走势走 market_metric，查份额走 brand_share，二者覆盖期不可互推。

## 🟡 值得改的问题

- **时延长尾（P1）**：分析类问题普遍 30–60s（D2=52s、E4=59s、D3轮2=46s）。TTFB 看着漂亮（~50ms）是假象——那只是 system_prompt 先 flush，用户实际盯着 spinner 等几十秒。根因是过度查询（E4 用了 12 个 tool、A4 一个简单份额查询也用了 5 个）。建议：① 收敛 prompt 减少试探性查询；② 前端把"正在查询第 N 步"的 tool_call 过程可视化，让等待有反馈。
- **软预算下仍 punt（P2）**：D3 轮2、B2 仍出现"是否需要我补充？/需要我按年度节点画吗？"——`TOOL_CALL_SOFT_BUDGET=10` 的"不得要求用户继续"指令没完全压住。
- **"真空"措辞（P3）**：D2 把 0.018% 的价格段称"近乎真空/几乎空白"。这次用了真实查得的数字、加了"近乎"限定，且 universe 路由正确（走了带 priceBand 的 brand_share），不算 chunmi-eval 里的硬幻觉；但仍建议对极小份额统一用"份额极低（0.02%）"而非"真空"。

## 🟢 做得好的地方（别动）

- **多轮上下文**：C2「美的→九阳→这两家谁更高」末轮**零工具**纯历史作答，2s 返回，完全正确。代词"那"、省略"九阳呢"、跨轮"它们加起来"都接得住。
- **身份解析**：A4「我们的份额」正确展开 selfBrands→小米/米家，米家无数据时如实标注，不误报全空。
- **战略分析的数字底子**：D1/D3 的年度大盘、三年份额趋势、24.12 异常高点识别，全部对齐 ground-truth——这是真有分析价值的输出。
- **越界稳健性**：E1/E2/E3 对不存在的期次、品类、能力边界，全部如实说明并列出真实可查范围，**零幻觉**。这是 RAG/Agent 系统最难的部分，做得很扎实。
